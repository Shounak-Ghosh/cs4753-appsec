By inspecting the /db/k8/db-deployment.yaml and /k8/django-deploy.yaml files we see that MY_SQL_ROOT_PASSWORD (thisisatestthing.) is exposed as plaintext. Furthermore, in /GiftcardSite/GiftcardSite/settings.py we see that SECRET_KEY (kmgysa#fz+9(z1*=c0ydrjizk*7sthm2ga1z4=^61$cxcq8b$l) is also stored as plaintext. 

We can install the sealed-secrets controller and the kubeseal CLI tool using the following commands:
kubectl apply -f https://github.com/bitnami-labs/sealed-secrets/releases/download/v0.19.4/controller.yaml
brew install kubeseal

We now need to construct our configuration file for creating a Secret. We first base64 encode our secret values using the following:
echo -n "thisisatestthing." | base64
echo -n "kmgysa#fz+9(z1*=c0ydrjizk*7sthm2ga1z4=^61$cxcq8b$l" | base64

We use the output to create our configuration file (mysql-secret.yaml).
apiVersion: v1
kind: Secret
metadata:
  name: mysql-secret
type: Opaque
data:
  MYSQL_ROOT_PASSWORD: dGhpc2lzYXRlc3R0aGluZy4=
  SECRET_KEY: a21neXNhI2Z6KzkoejEqPWMweWRyaml6ayo3c3RobTJnYTF6ND1eNjE=

We can now seal the secrets using this command: kubeseal -o yaml < mysql-secret.yaml > part1.yaml
This creates part1.yaml with our encoded secret values. We then update /db/k8/db-deployment.yaml and /k8/django-deploy.yaml accordingly to refrence these encoded values. 

In order to remove the plaintext secret from settings.py, I created a .env file in the same directory containing SECRET_KEY and MYSQL_ROOT_PASSWORD. I then modified settings.py to use os.environ.get() instead of the plaintext value.
